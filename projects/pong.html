Pong - Target Practice

Project Overview

This is a small game that I decided to make as a solo developer in only the one day that I was available for the Alberta Game Jam. This is essentially Pong, except instead of competing against the opposite paddle, you work together to aim the ball and try to hit a moving target in the arena.

Development Notes

I wanted to make a game without an established Game Engine in C++ since I have never gone game-engineless before, and I've been wanting to brush up on my C++.

I used functions from the standard library and SFML for a graphics library. Everything else was coded by me. Naturally with the tight timeline, it took the classic game jam pipeline of high hopes of clean code, quickly turning into the panic of finishing on time as a big ball of spaghetti emerges. Nonetheless I am pretty proud of how it turned out considering the circumstances.

As an added challenge for myself, I did not allow myself to google anything other than C++ syntax information and compilation/linking errors I was unfamiliar with.

My Reflections

  • I had a lot more fun than I expected making this. The satisfaction of making shapes render and move at a consistent velocity was incredibly satisfying, and building the architecture yourself really helps understand how things work under the hood.
  • Going engineless isn't necessarily as intimidating as I thought it might be, but it is a lot more time consuming just to get basic functionality.
