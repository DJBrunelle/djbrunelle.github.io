2.5D Puzzle

Project Overview

This is a prototype for a puzzle game in which you're navigating a grided cube maze. You can only see in 2 dimensions at a time, but you can switch between the XY-XZ planes. by switching viewing planes, you can see different parts of the cube.

This was inspired by 4D Minecraft - you navigate the world in 3 dimensions, but there's a 4th axis which you can switch between which allows you to discover a whole new world by only rotating that conceptual 4th axis. This is just taking away one dimension so it's easier for humans to comprehend.

Development Notes

The transition effect was really fun to work on. This project was an intro for me into how event systems can really work well in games. In this case, an event fires every time the transition button is pressed, and all relevant game objects that need to respond will know to do so without tight coupling. This type of system has a lot of power, but I can see how it would become very difficult to debug if you take it too far, as it's harder to tell what is responding to what.

This project was also great at helping me improve my basic linear algebra skills and cognitive map abilities. Keeping track of axes and programming such that the player always snaps to the nearest gridpoint was a great exercise.
